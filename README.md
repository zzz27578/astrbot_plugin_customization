# AstrBot 欢迎私聊定制插件

新人加入群聊后，机器人自动私聊发送三条欢迎素材：

1. QQ JSON 卡片
2. 合并转发形式的聊天记录
3. 图片

插件支持 AstrBot `aiocqhttp` / OneBot v11 平台，适合 NapCat、Lagrange 等协议端。它还提供一个 AstrBot 插件页面，用于管理素材、启用群、发送策略、失败兜底、异常通知和最近日志。

## 核心能力

- 监听 OneBot `group_increase` 新人入群事件。
- 自动私聊新人，不需要在群内触发命令。
- QQ 卡片保存原始 `json` 消息段，发送时尽量保持 QQ 原生卡片样式。
- 聊天记录使用 OneBot `send_private_forward_msg` 发送合并转发消息。
- 新人非好友时，发送会携带来源 `group_id`，让 NapCat/Lagrange 尽量按群临时会话发送。
- 图片支持 QQ 引用采集和 WebUI 上传。
- 支持所有群启用、白名单群启用、全部关闭，以及群黑名单排除。
- 支持发送队列、重试、去重、降级文本、管理员 QQ 通知、管理群通知和群内兜底。

## 安装

将本仓库放入 AstrBot 的插件目录：

```bash
cd AstrBot/data/plugins
git clone https://github.com/zzz27578/astrbot_plugin_customization.git
```

重载或重启 AstrBot 后，在插件详情页打开 **欢迎私聊工作台**。

## 首次配置流程

1. 把 QQ 卡片发给机器人所在的群或私聊。
2. 回复这张卡片发送：`/卡片 添加 内部群邀请`
3. 回复一条希望作为聊天记录的消息发送：`/记录 添加 默认记录`
4. 回复图片发送：`/图片 添加 默认图片`，或在插件页面上传图片。
5. 在插件页面确认三条素材均已启用。
6. 使用 `/测试 你的QQ号` 验证实际私聊效果。
7. 按需要设置群白名单、黑名单、失败兜底和通知对象。

## 内置指令

所有指令都注册为 AstrBot 标准命令，不再需要 `/欢迎` 前缀。请按你的 AstrBot 命令前缀使用，常见形式是 `/状态`、`/卡片 添加 名称`。

插件命令进入 AstrBot command filter，一般不会作为普通聊天交给 LLM 思考。一般直接发送下面任一形式：

```text
/状态
/测试 123456
/card add 内部群邀请
```

### 状态与测试

```text
/帮助
/状态
/测试 [QQ]
```

- `状态`：查看总开关、启用模式、当前素材、群名单和管理员 QQ。
- `测试`：向指定 QQ 发送当前启用的三条欢迎素材。不填 QQ 时发送给命令发送者。

### 群控制

```text
/模式 all
/模式 whitelist
/模式 disabled
/启用 当前群
/启用 群号
/禁用 当前群
/禁用 群号
```

模式说明：

- `all`：所有群启用，黑名单群排除。
- `whitelist`：仅白名单群启用。
- `disabled`：全部关闭。

### QQ 卡片

```text
/卡片 添加 名称
/卡片 使用 名称
/卡片 列表
/卡片 删除 名称
```

添加卡片时，需要回复 QQ 原生卡片消息。插件会读取引用消息中的 `json` 消息段并保存原始 JSON 字符串。

### 聊天记录

```text
/记录 添加 名称
/记录 使用 名称
/记录 列表
/记录 删除 名称
```

添加聊天记录时，需要回复一条消息。插件会把被回复消息保存为合并转发节点。同名模板会继续追加节点，因此可以多次回复不同消息，把它们组成一份合并转发聊天记录。

如果回复的是一条已经存在的 QQ 合并转发聊天记录，插件会尝试调用 `get_forward_msg` 展开内部节点并直接保存，避免发送时再套一层“bot 名称：[聊天记录]”外壳。

### 图片

```text
/图片 添加 名称
/图片 使用 名称
/图片 列表
/图片 删除 名称
```

添加图片时，需要回复图片消息。也可以在插件页面上传本地图片，上传后会自动设为当前启用图片。

### 管理员与通知

```text
/管理员 添加 QQ
/管理员 删除 QQ
/通知 开
/通知 关
/群内兜底 开
/群内兜底 关
```

- 管理员 QQ 可直接使用本插件管理命令。
- `通知` 控制是否私聊管理员 QQ 发送失败提醒。
- `群内兜底` 控制私聊失败时是否在原群提醒新人。

## 插件页面

插件详情页中的 **欢迎私聊工作台** 包含：

- 概览：总开关、队列、当前启用素材、测试发送。
- 素材库：卡片、聊天记录、图片的启用、重命名、删除和上传。
- 群控制：启用模式、白名单群、黑名单群、管理员 QQ。
- 发送策略：发送顺序、消息间隔、重试、去重。
- 失败兜底：卡片/聊天记录/图片降级文本、群内兜底。
- 通知日志：管理员私聊通知、管理群通知、成功通知和最近发送记录。

## 稳定性与兜底

主链路会把新人入群事件放入发送队列，按顺序私聊三条消息，避免并发发送导致风控风险。

新人入群依赖 NapCat/Lagrange 等 OneBot 协议端向 AstrBot 上报 `notice_type=group_increase` 事件。如果页面日志没有任何入群记录，请先检查协议端是否开启/正常上报 notice 事件，而不是先检查命令。

可选兜底包括：

- 私聊失败重试。
- 卡片失败后发送备用文本或链接。
- 聊天记录失败后发送备用文本。
- 图片失败后发送备用文本。
- 失败后私聊管理员 QQ。
- 失败后发送到管理群。
- 私聊全部失败或任一消息失败时，在原群 @ 新人提醒。
- 同群同人去重，避免反复退群进群重复触发。

需要注意：QQ 隐私设置、账号风控和协议端限制可能导致私聊失败。插件可以重试、降级和提醒，但不能绕过 QQ 平台本身的限制。

## 数据保存

插件数据保存在 AstrBot 的 `data/plugin_data/astrbot_plugin_customization/` 下，包括：

- `store.json`：素材、设置、去重和日志。
- `images/`：插件页面上传的本地图片。

数据不会写入插件代码目录，更新插件时不应覆盖已有素材和配置。

## 兼容性

- AstrBot：建议 `>=4.17.0`
- 平台：`aiocqhttp`
- 协议：OneBot v11

QQ JSON 卡片在不同协议端和 QQ 版本中可能有差异。如果原始 JSON 卡片被协议端拒绝，建议配置备用文本或备用链接。
