# AstrBot 欢迎私聊工作台

开发者：百岁老太
联系方式：QQ：2950506809

新人进入群聊后，机器人会自动向新人私聊发送当前配置的内容：

1. QQ 卡片
2. 群聊天记录
3. 图片
4. 文字

这不是普通欢迎语，而是一套可控、可审计、可兜底的入群私语系统。适合多个群同时使用，也适合需要统一新人引导、入群说明、资源入口或身份验证提示的场景。

## 安装

进入 AstrBot 插件目录：

```bash
cd AstrBot/data/plugins
git clone https://github.com/zzz27578/astrbot_plugin_customization.git
```

重载或重启 AstrBot，然后在插件详情页打开 **欢迎私聊工作台**。

## 首次配置

1. 在 QQ 中把需要保存的卡片、聊天记录、图片准备好。
2. 回复 QQ 卡片，发送 `/卡片 添加 名称`。注：恢复指的是左右滑动消息并引用恢复
3. 回复聊天记录或消息，发送 `/记录 添加 名称`。
4. 回复图片，发送 `/图片 添加 名称`，也可以在插件页面上传图片。
5. 在插件页面的素材库填写文字内容。每一行会作为一段文字发送，并可在 **发送策略** 中勾选文字。
6. 在 **群控制** 中选择启用范围。千人大群建议先使用白名单。
7. 在 **发送策略** 中设置间隔、重试、去重时间。
8. 用 `/测试 QQ号` 验证新人实际收到的当前配置。

## 工作台页面

**概览**

查看总开关、发送队列、当前启用的卡片、聊天记录和图片，也可以进行测试发送。

**素材库**

管理 QQ 卡片、聊天记录、图片和文字。卡片、聊天记录、图片可以启用、重命名、删除；文字直接在页面中填写，分行表示分段发送。

**群控制**

设置全部群启用、白名单群启用、全部关闭，以及群黑名单。

**发送策略**

设置消息发送顺序、消息间隔、失败重试、同群同人去重。

**送达复核与补偿** 专门处理 NapCat 的 `NTEvent` 空错误超时。插件会先等待一段时间，尝试扫描最近私聊历史；如果没有确认到消息，才按限制次数补发。这个能力用于减少漏发，不会无限重试，也不是 QQ 官方已读回执。

这里还提供 **AstrBot 工具调用轮数上限**。它用于调整 AstrBot Agent 的工具调用上限；新人私聊本身由插件队列执行，不依赖 LLM 思考。

**失败兜底**

卡片、聊天记录、图片发送失败时，可以分别发送备用文本。

如果某一项没有配置，且没有可用兜底文本，该项会自动跳过，不会阻碍测试或新人入群发送。

群内兜底默认关闭。开启后可选择触发条件：

- 全部私聊失败
- 任一消息失败
- 有新人进入

`@ 新人` 默认关闭，避免对群内造成过多打扰。

**通知日志**

设置管理员 QQ、管理群通知、成功通知、日志保留条数。最近记录支持一键清空。

## 稳定性说明

多人同时进群时，插件会把新人加入事件放入发送队列，按顺序处理，避免同一时间对 QQ 协议端发起过多私聊请求。

大群推荐配置：

- 使用白名单控制先上线哪些群。
- 保持同群同人去重开启，默认 1440 分钟。
- 消息间隔不要设为 0，建议 1.5 秒或更高。
- 保留失败重试，但重试次数不要过高。
- 开启送达复核与补偿，补发次数建议保持 1 次。
- 群内兜底保持关闭，确认私聊稳定后再按需开启。
- 开启管理员通知，便于第一时间看到失败原因。

这样可以同时兼顾稳定发送和防止对新人造成垃圾信息骚扰。

## 常见问题

**新人没有收到私聊**

先看通知日志是否出现记录。若没有记录，检查群是否在启用范围内，以及协议端是否正常上报新人进群事件。

**提示需要先添加好友**

请确认机器人在该群内拥有足够权限，并使用群临时会话发送。插件会携带来源群信息，但最终能否送达仍取决于当前 QQ 账号、群权限和协议端行为。

**聊天记录外面又套了一层**

重新引用原始聊天记录执行 `/记录 添加 名称`。插件会尽量展开原始聊天记录，避免发送时再包一层机器人名称。

**不想打扰群内成员**

保持群内兜底关闭，且不要开启 `@ 新人`。默认设置就是低打扰模式。
